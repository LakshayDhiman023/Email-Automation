Principles

Six rules that don't bend

Every feature request is tested against these. If it breaks one, it doesn't ship.

P1

Free forever for one user

Every dependency has a $0 tier — Supabase, Gmail API, free hosts, external cron.

P2

Self-hosted & owned

Your database, your OAuth app, your tokens. Single-tenant by design — "multi-user" means another clone.

P3

Human in the loop

Nothing sends without explicit approval. pending → approved → sent is the heart of the product.

P4

Deliverability is sacred

Daily cap, suppression list, cooldowns, bounce detection, sends only in the recipient's working hours.

P5

Respect the channel

No scrape-and-spray, no tracking pixels, no ToS-violating LinkedIn/X/Instagram automation.

P6

Learning-grade code

Readable FastAPI + React with real tests, migrations, and CI — the repo doubles as a portfolio.

Roadmap

From working product to platform

v0.1 already sends, follows up, detects replies, and protects your Gmail reputation. What's next is polish, intelligence, then community.

SHIPPEDv0.1 — the single-player loop
  • Dynamic {variable} templates with auto-generated compose forms
  • Approval pipeline: edit, approve, cancel, close threads
  • Configurable timezone, send windows, working days, holidays
  • Reply detection, OOO rescheduling, bounce → auto-suppression
  • Safety rails: 40/day cap, 30-day cooldown, hard opt-out list
  • Runs on sleeping free hosts (external cron drives the work)
  • OWASP-hardened API · 29 tests · CI · Dropbox-language UI
NEAR · v0.2Polish the loop
  • One-command setup clone → running in 15 minutes
  • Published OAuth app guide stop the 7-day token expiry
  • Multi-step follow-up sequences today: one follow-up max
  • Inline reply reading in the dashboard
MID · v0.3The "wow" layer — still $0
  • AI reply drafting via local Ollama drafts in your voice; still human-approved (P3)
  • Template analytics & A/B compare reply rate per template
  • CSV contact import with variable mapping
  • Daily digest of replies and pending approvals
LONG · v1.0A platform others build on
  • Shareable "campaign packs" job hunt, fundraising, user research
  • Provider abstraction Outlook / SMTP alongside Gmail
  • Optional login for public deployments
  • One-click deploys + docs site
  • Never: DM automation, tracking pixels, 1,000-email blasts, SaaS billing
